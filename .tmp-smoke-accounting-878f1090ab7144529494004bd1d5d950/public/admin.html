URL, auth va ffmpeg yordam

snapshotUrl odatda kameraning JPG snapshot endpointi bo'ladi. Ko'p Hikvisionlarda http://IP/ISAPI/Streaming/channels/101/picture, ko'p Axislarda http://IP/axis-cgi/jpg/image.cgi ishlaydi.

streamUrl RTSP yoki HTTP MJPEG bo'ladi. RTSP ko'p hollarda rtsp://IP:554/..., HTTP MJPEG esa http://IP/.../mjpg yoki .../video.cgi ko'rinishida bo'ladi. Hozir server MJPEGning birinchi JPEG frameini ham o'qiy oladi.

basic oddiy login/parol bilan LAN kameralar uchun. bearer ko'proq cloud API yoki VMS gateway uchun. query token URL ichiga qo'shiladigan servislarda ishlatiladi. Oddiy zavod IP kamerada odatda basic yetadi.

ffmpegPath bo'sh qolsa ham bo'ladi. Server ichida paket bilan kelgan ffmpeg ishlatiladi; faqat o'zingiz alohida ffmpeg qo'ymoqchi bo'lsangiz yo'l kiriting.

URLni topish joyi: kamera web paneli, vendor manuali, ISAPI, VAPIX, RTSP, ONVIF, Live View, Integration bo'limlari.

O'rnatish tavsiyasi

Eng yaxshi count uchun kamerani lentaga imkon qadar tikroq, odatda 75°-90° yuqoridan qaratish tavsiya qilinadi.

Agar tik joylashtirish imkoni bo'lmasa, 35°-45° qiya burchakdan faqat bitta qatorda, bricklar bir-birini yopmaydigan oqimda ishlating.

Kamera chiziq ustidan 2.8-4.5 metr balandlikda, tebranishsiz kronshteynda va soya tushmaydigan bir xil yoritishda bo'lsa aniqlik ancha oshadi.
